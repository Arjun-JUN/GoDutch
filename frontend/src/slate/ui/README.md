# frontend/src/slate/ui

> shadcn/ui primitive components: accessible, unstyled base components used by Slate's branded layer.

## Overview

This directory is a local copy of shadcn/ui components (Dialog, Sheet, Tabs, Card, Button, Input, Form, Popover, etc.). They are generated by the shadcn CLI and live in the repo as editable source, not `node_modules`. That means they can be customized, but doing so means diverging from upstream — keep customizations minimal.

Pages should not import directly from this directory. All consumption should go through `../components/`.

## How it works

shadcn/ui components use Radix UI primitives under the hood for accessibility (focus trap, ARIA, keyboard nav) and `cn()` (clsx + tailwind-merge) for class composition. No extra wiring needed — just import and use within `components/`.

## Key files

~40 components including: `button.jsx`, `input.jsx`, `dialog.jsx`, `sheet.jsx`, `card.jsx`, `form.jsx`, `tabs.jsx`, `popover.jsx`, `select.jsx`, `badge.jsx`, `avatar.jsx`, `tooltip.jsx`, `calendar.jsx`, `sonner.jsx` (toast).

## Interactions

| Direction | Who | How |
|-----------|-----|-----|
| Upstream | `../components/` | Import and wrap with GoDutch branding |
| Downstream | `@radix-ui/*` | Accessibility primitives |
| Downstream | `class-variance-authority` | Variant styling |

## Gotchas

- Do not import these directly in pages. The abstraction layer in `components/` is intentional.
- When running `npx shadcn add <component>`, the new file lands here. Review it for Tailwind class conflicts with GoDutch tokens before using.
